# VideoGrace Дорожная карта 2024

### Стратегическое видение

Цель - превратить систему Videograce в глобальное распределенное средство коммуникации с тысячами узлов. Узлы могут принадлежать как ООО "ИВС", так и независимым специалистам или компаниям.

## Backend

Основной акцент развития системы направлен на backend. Сервер должен стать узлом распределенной сети, обеспечивая связность для пользователей, подключенных к разным серверам. Также необходимо предоставить оптимальное WebSocket API для удобства создания легких JS-клиентов для браузера.

### Новый Web UI админки сервера

 - Использование REST для всего функционала сервера
 - Система управления токенами
 - Внедрение системы webhook/sh hook для интеграции с внешними системами
 - Внедрение системы ограничения прав для админа сервера (для облачного применения)
 - Создание BLOB хранилища
 - Введение Node integration API
 - Подготовка подробной документации

## Клиентские приложения

Приоритетом развития клиентской части, должно стать создание максимально удобного и интуитивно понятного интерфейса пользователя.

### Desktop
 - Многострочный input
 - Автоматическое переподключение после потери связи
 - Панель событий
 - Поддержка многомониторной конфигурации
 - Использование смайлов
 - Обмен файлами, изображениями, BIO
 - Возможность множественного входа
 - Поддержка MacOS
 - Эффектные тени и сглаживание
 - Создание подробной документации

### Web
 - Разработка MVP на JS

### Android
 - Разработка MVP

### iOS
 - Разработка MVP

## SDK Работы

Приоритет в разработке SDK - создание логичного инструмента с минимальным оверхедом. Документация должна быть разработана _перед_ кодированием. Создание минимальных примеров приложений должно предшествовать созданию функциональности в официальных клиентах.

### Работы
 - Создание документации
 - Разработка примеров ботов, приложений и интеграционных скриптов
 - Разработка биндингов для других языков (C, Python, JS)

## Облачное направление

Облачное направление требует тестирования гипотез и реализации дополнительного слоя безопасности.

### Работы
 - Интеграция системы биллинга с admin API сервера
 - Балансировка и горизонтальное масштабирование
